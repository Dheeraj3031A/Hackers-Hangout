HTTP

  • Underlying protocol of the WWW
  • Developed by Tim Berners-Lee & his team Between 1989-1991
  • Transfer data in the form of plan text, hypertext audio, video etc..
  • Known as Hypertext Transfer Protocol because of its efficiency that allows us to use in a hypertext environment where there are rapid jumps between one document to another
  • Similar to FTP because they both transfer data from host to host HTTP is simpler because it uses only one connection
  • Similar to SMTP because they transfer data between client and server Difference is that HTTP does not store the data and the response is sent immediately unlike SMTP

Features of HTTP

  • Connection-less protocol

    • It initiates a request and waits for a response from the server, when received, it sends thee response back after which the client disconnects the connection. The connection only exists during the request and response stage

  • Media Independent

    • It is a media independent because data can be sent as long as both the client and server knows how to handle the data content. Required for both the parties to specify the content type in request and response header

  • Stateless

    • It is stateless as both the client and the server knows each other only during the ongoing connection after which they have no connection and does not retain any information that was transferred

  • Request Methods

    • HTTP defines different request methods that indicate the intended action to be performed on a resource. The commonly used methods are GET (retrieve a resource), POST (submit data to be processed), PUT (update a resource), DELETE (remove a resource), and more. These methods allow clients to interact with resources on the server in a standardized manner.

  • Uniform Resource Identifiers (URIs)

    • HTTP uses URIs to identify and locate resources on the web. URIs typically take the form of URLs (Uniform Resource Locators) and provide a way to uniquely address resources such as web pages, images, APIs, and more.

  • Status Codes

    • HTTP uses status codes to indicate the outcome of a request. These codes are three-digit numbers sent by the server in the response header. They provide information about whether the request was successful, redirected, encountered an error, or other status-related details. Examples of status codes include 200 OK, 404 Not Found, 500 Internal Server Error, and many more.

HTTP Versions

  • It uses the <major>.<minor> numbering scheme to indicate version of the protocol
    • The commonly used methods are GET (retrieve a resource), POST (submit data to be processed), PUT (update a resource), DELETE (remove a resource), and more. These methods allow clients to interact with resources on the server in a standardized manner.
  • The syntax is “HTTP” “/” Major.Minor
  • There are 5 versions
    • HTTP/0.9, HTTP/1.0, HTTP/1.1, HTTP/2.0 and HTTP/3.0
  • Today the version in common use is HTTP/1.1, HTTP/2.0 and HTTP/3.0
