HTTP

  • Underlying protocol of the WWW
  • Developed by Tim Berners-Lee & his team Between 1989-1991
  • Transfer data in the form of plan text, hypertext audio, video etc..
  • Known as Hypertext Transfer Protocol because of its efficiency that allows us to use in a hypertext environment where there are rapid jumps between one document to another
  • Similar to FTP because they both transfer data from host to host HTTP is simpler because it uses only one connection
  • Similar to SMTP because they transfer data between client and server Difference is that HTTP does not store the data and the response is sent immediately unlike SMTP

Features of HTTP

Connection-less protocol

It initiates a request and waits for a response from the server, when received, it sends thee response back after which the client disconnects the connection. The connection only exists during the request and response stage

Media Independent

It is a media independent because data can be sent as long as both the client and server knows how to handle the data content. Required for both the parties to specify the content type in request and response header

Stateless

It is stateless as both the client and the server knows each other only during the ongoing connection after which they have no connection and does not retain any information that was transferred

Request Methods

HTTP defines different request methods that indicate the intended action to be performed on a resource. The commonly used methods are GET (retrieve a resource), POST (submit data to be processed), PUT (update a resource), DELETE (remove a resource), and more. These methods allow clients to interact with resources on the server in a standardized manner.

Uniform Resource Identifiers (URIs)

HTTP uses URIs to identify and locate resources on the web. URIs typically take the form of URLs (Uniform Resource Locators) and provide a way to uniquely address resources such as web pages, images, APIs, and more.

Status Codes

HTTP uses status codes to indicate the outcome of a request. These codes are three-digit numbers sent by the server in the response header. They provide information about whether the request was successful, redirected, encountered an error, or other status-related details. Examples of status codes include 200 OK, 404 Not Found, 500 Internal Server Error, and many more.

HTTP/0.9

  • Client-Server, Request-Response, TELNET-friendly protocol
  • Method + path for the resource
  • Only GET requests
  • Response was Hypertext only
  • Terminated immediately after response
  • No HTTP Headers, only text contents can be transferred and no status codes
  • No URL, No versioning

HTTP/1.0

  • Browser Friendly
  • Introduction of headers which provided metadata about request & response(HTTP Version numbers, status codes, content type)
  • Ability for the server to respond with not only Hypertext but also other files such as media, scripts etc..
  • GET, POST, HEAD Methods were supported
  • Connection terminated immediately after response

HTTP/1.1

  • Commonly used HTTP version
  • Performance optimization & feature enhancements. Added features like Persistent and Pipeline connections, chunked transfers, compression/decompression, virtual hosting etc.
  • Faster response time and bandwidth saving using cache
  • Supported methods include GET, HEAD, POST, PUT, DELETE, TRACE, OPTIONS
  • With persistent connections, after a client sends a request and receives a response from the server, the connection remains open. This allows subsequent requests to be sent over the same connection without the need to establish a new connection for each request

HTTP/2.0

  • HTTP/2.0 is an updated version of the HTTP protocol.
  • Its goal is to improve website performance and reduce latency.
  • It introduces multiplexing, allowing simultaneous requests and responses over a single connection.
  • Header compression is implemented to reduce data transmission size.
  • Server push enables proactive sending of resources to improve page loading times.
  • HTTP/2.0 optimizes network resource usage for faster data transfer.
  • It requires support from both websites and browsers.
  • Most modern browsers and servers have added support for HTTP/2.0.

HTTP/3.0

  • HTTP/3.0 is the latest version of the HTTP protocol.
  • It is based on the QUIC transport protocol, which stands for Quick UDP Internet Connections.
  • The goal of HTTP/3 is to further enhance web performance and security.
  • It addresses limitations of TCP by utilizing UDP as the underlying transport protocol.
  • UDP allows for faster and more reliable data transmission compared to TCP.
  • HTTP/3 introduces stream multiplexing, enabling multiple data streams to be sent over a single connection simultaneously.
  • Improved congestion control mechanisms are implemented in HTTP/3 to better manage network traffic.
  • HTTP/3 includes built-in encryption, enhancing security for data transmission.
  • The use of HTTP/3 results in improved latency, meaning faster response times.

  • HTTP/3 reduces packet loss, leading to more reliable data transmission.
  • Overall, HTTP/3 offers a better web browsing experience for users through improved performance and enhanced security.

HTTP Methods

GET

The GET method is used to retrieve or fetch data from a server. When you enter a website URL in your browser and press enter, it sends a GET request to the server to retrieve the webpage and display it in your browser.

HEAD

The HEAD method is similar to GET, but it only requests the headers of a resource, such as the content type, last modified date, or server information. It does not retrieve the full content of the resource. The HEAD method is often used to check if a resource has been modified or to gather information about a resource without downloading it.

PUT

The PUT method is used to upload or replace a resource on the server. It sends the entire representation of the resource to the specified URL. It can be used to create a new resource or update an existing one.

POST

The POST method is used to submit data to the server to be processed. It is commonly used when submitting forms on websites. When you fill out a form and click the submit button, the data you entered is sent to the server using a POST request.

DELETE

The DELETE method is used to delete a specified resource on the server. It sends a request to delete the resource at the specified URL.

CONNECT

The CONNECT method is used to establish a network connection to a server over a proxy. It is typically used for creating secure tunnels or for enabling communication through proxies.

OPTIONS

The OPTIONS method is used to retrieve the supported methods, headers, or capabilities of a server. It allows a client to check what operations are available on a particular resource.

TRACE

The TRACE method is used for diagnostic purposes. It sends a request to the server, which then reflects the received message back to the client. It helps in debugging or tracing the path of a request through intermediate servers.

HTTP State Retention

HTTP state retention is the ability of a web server to remember the client's state after a request is completed. It can enhance the user experience by allowing clients to pick up where they left off without starting over. Common methods include using cookies or session IDs to store information.

Downsides of HTTPS

  • Need to purchase SSL Certificates
  • Web applications load slower compared to HTTP
  • Have to update all the links inside a website to https

Benefits of HTTP state retention

  • Improved user experience: Clients can seamlessly interact with a web application without frequent login requirements.
  • Reduced bandwidth usage: Less data needs to be sent from the server to the client, saving bandwidth.

Risks of HTTP state retention

  • Security risks: Inadequate security measures can lead to cookie or session ID theft, allowing attackers to impersonate clients.

What are Cookies?

  • Security risks: Inadequate security measures can lead to cookie or session ID theft, allowing attackers to impersonate clients.
  • Privacy risks: Excessive collection of client information may compromise privacy.
  • They allow websites to recognize you when you revisit them and tailor the results to your preferences.
  • Cookies have a long history in internet usage and have evolved significantly over time.

Types of Cookies

Session Cookies

  • Session cookies are temporary cookies that are stored in your browser's memory during your browsing session.
  • They are used to remember your actions and preferences on a website as you navigate through different pages.
  • Session cookies are automatically deleted when you close your browser or end your browsing session.

Persistent Cookies

  • Persistent cookies are stored on your device even after you close your browser or end your browsing session.
  • They have an expiration date set by the website and remain on your device until that date or until you manually delete them.
  • Persistent cookies are used to remember your preferences and settings for future visits to the website.

Secure Cookies

  • Secure cookies are encrypted cookies that are transmitted over a secure (HTTPS) connection.
  • They provide an extra layer of protection for sensitive information, such as login credentials or financial data.
  • Secure cookies help prevent unauthorized access and protect your privacy during data transmission.

Zombie Cookies

  • Zombie cookies are a type of persistent cookie that can be automatically recreated even after being deleted.
  • They use various techniques, such as browser vulnerabilities or backup files, to revive themselves.
  • Zombie cookies can be used for tracking purposes without your consent and can be challenging to remove completely.

Need for cookies

  • Cookies were created to improve user experience by providing developers with information about the user.
  • They can remember language preferences on websites, eliminating the need for repeated selections.
  • By saving preferences as cookies, the server can recognize them when the user visits again.
  • Cookies now serve multiple purposes beyond language settings, enhancing various aspects of web browsing and customization.

Advancement for cookies

  • Initial Purpose: Enhancing user experience by saving small amounts of information as text on the client's machine.
  • Expanded Usage: Usage of cookies significantly increased, encompassing functions like storing search history and shopping cart contents.
  • Challenges with Volume: The growing volume of stored information in cookies led to larger sizes and potential performance issues on the client's machine.
  • Alternative Approaches: Developers responded by devising alternative methods to manage cookies, avoiding excessive data storage on the client's machine.

Security threats related to Cookies

Cookies saves your information to your machine. This information is highly personal to you and should only be retrieved by the server which saved it.

The following things should be kept in mind:

  • Cookie server mapping:: A cookie is specific to the website that created it and cannot be used by any other server. The ID saved in the cookie is directly linked to the website's database.
  • Limited access to system information: Cookies cannot access any information saved on your system, browser, or hard disk. They are limited to the data stored by the website that created them.
  • No threat of viruses or threats: Cookies do not pose a threat in terms of delivering viruses or any other malicious software. They are simply small text files used for storing information.
  • Permission-based information access: With appropriate permissions, a third-party website can utilize a cookie to access the information and/or preferences stored within it. This can be done for both the website's benefit and the user's benefit.

Hypertext Transfer Protocol Cache

  • HTTP caching involves storing commonly or frequently used data in a location that is quick to access.
  • The purpose of caching is to improve the speed and efficiency of data retrieval.
  • By storing frequently accessed data closer to the requesting computer, it reduces the need to fetch the data from a remote location.
  • Caching enables quicker access to data because the computer doesn't have to reach too far to retrieve it.
  • With HTTP caching, there is a higher probability of accessing the needed data faster, resulting in improved performance.
  • Caching is particularly beneficial for repeated requests for the same data, as it eliminates the need to retrieve it multiple times.
  • HTTP caching is commonly used in web browsers and servers to optimize the delivery of static resources, such as images, CSS files, and JavaScript files.

Cache Headers in HTTP

HTTP Caching has two major cache headers

  • Cache-Control
  • Expire

Cache-Control

  • Cache-Control acts as a switch to enable caching in the user's browser.
  • When this header is included, it enables caching for supported web browsers.
  • If Cache-Control is absent, web page contents won't be cached, even if the browser supports caching.

Cache-Control has two privacy settings

Public

Resources can be cached by intermediate proxies, like Content Delivery Networks (CDNs).

Private

Caching is limited to a single user and not performed by intermediate proxies.

Expires

  • Expires header is used in conjunction with Cache-Control.
  • It specifies a date from which cached resources are considered invalid.
  • When the cache expires, the user's web browser will request all page content again.

THE GENERATIONS OF WEB

Web 1.0

Limited content creators

During Web 1.0, there were only a few individuals or organizations creating content for the web. This resulted in a relatively small number of websites available.

Majority of users were consumers of content

In Web 1.0, the vast majority of users were passive consumers of the content created by the limited number of content creators. Users mainly accessed static web pages and consumed the information presented to them.

Web 2.0

User-generated content

Web 2.0 introduced a shift towards user-generated content, where individuals could actively contribute and create content on the web. This led to the emergence of platforms that encouraged user participation, such as social media, blogs, and wikis.

Emphasis on usability and interoperability

Web 2.0 focused on improving the usability of websites and enhancing the ability of different systems and platforms to work together seamlessly. This resulted in user-friendly interfaces, intuitive navigation, and the integration of various web technologies.

Adoption of technologies like AJAX and JavaScript frameworks

Web 2.0 saw the adoption of advanced technologies like Asynchronous JavaScript and XML (AJAX) and JavaScript frameworks. These technologies enabled more dynamic and interactive web experiences, allowing for real-time updates and enhanced user interactivity.

Greater user engagement and participation

With the introduction of Web 2.0, users became active participants in the web ecosystem. They could comment, share, collaborate, and contribute their own content, fostering a sense of community and engagement.

Web 3.0

Integration of Distributed Ledger Technology (DLT) like blockchain

Web 3.0 incorporates technologies like blockchain, which enable decentralized and secure data storage and transactions. This allows for increased privacy, transparency, and trust in online interactions.

Transformation of the web into a database

Web 3.0 aims to transform the web from a collection of static pages into a vast interconnected database. This shift allows for easier data discovery, retrieval, and analysis, enabling more personalized and intelligent web experiences.

Smart Contracts based on individual needs

Web 3.0 introduces the concept of Smart Contracts, which are self-executing contracts with predefined rules and conditions. These contracts can be tailored to meet individual needs and automatically execute actions based on specific triggers or events.

Focus on backend development and data sharing

Web 3.0 emphasizes backend development and the efficient sharing of data. It aims to enable seamless integration and interoperability across various platforms, applications, and services.

Different perspectives and services for the same web and data

Web 3.0 allows for different services and applications to access and present data from various perspectives, catering to individual preferences and requirements. This enhances user experiences by providing customized views and personalized interactions.

Organizing information in a structured manner

BIG DATA – A Special Discussion

Big Data refers to complex and large data sets that have to be processed and analysed to uncover valuable information that can benefit businesses and organisations.

Structured Data

  • Structured data is one of the types of big data.
  • It refers to data that has a fixed format and can be easily processed, stored, and retrieved.
  • Structured data is highly organized and can be stored in databases, allowing for simple search algorithms to access it.
  • Examples of structured data include tables in databases, where information is organized in a specific manner, such as employee details, job positions, and salaries.

Unstructured Data

  • Unstructured data is another type of big data.
  • It refers to data that lacks a specific form or structure.
  • Processing and analyzing unstructured data can be challenging and time-consuming.
  • Examples of unstructured data include emails, social media posts, audio and video files, and text documents without a predefined format.

Semi-structured Data

  • Semi-structured data is the third type of big data.
  • It contains elements of both structured and unstructured data.
  • Although not classified under a specific repository or database, semi-structured data includes tags or markers that help segregate individual elements within the data.
  • Semi-structured data may have some level of organization, but it doesn't conform to a rigid structure like structured data.

Characteristics of Big Data

Variety

  • Variety refers to the different types and sources of data that constitute big data.
  • Big data encompasses structured, unstructured, and semi-structured data.
  • Data is collected from various sources such as spreadsheets, databases, emails, PDFs, photos, videos, audio, and social media posts.
  • The variety of data poses challenges for processing and analyzing information from diverse sources.

Velocity

  • Velocity represents the speed at which data is being generated and collected in real-time.
  • It encompasses the rate of change, linking of incoming data sets at varying speeds, and bursts of activity.
  • Data received and acted upon in real-time is a critical aspect of velocity.
  • Some applications or systems require real-time evaluation and processing of data, especially for internet-based smart products.

Volumes

  • Volume is a key characteristic of big data.
  • It refers to the large amount or scale of data being generated on a daily basis.
  • Data is generated from various sources such as social media platforms, business processes, machines, networks, and human interactions.
  • The massive volume of data requires storage in data warehouses or specialized systems capable of handling and processing such vast amounts of information.

Web information retrieval (WebIR) is the process of retrieving relevant information from the World Wide Web. It is a sub-field of information retrieval (IR) that deals with the challenges posed by the Web's massive size, dynamic nature, and heterogeneity

WEB INFORMATION RETRIEVAL TOOLS

These are automated methods for retrieving information on the Web and can be broadly classified as search tools or search services.

Search Tools

  • Search tools are user interfaces that allow users to specify queries and browse the results.
  • They employ "robots" to index web documents, enabling efficient search functionality.
  • The search engine is the core component responsible for searching the index and retrieving relevant documents.

    • Search tools can be classified into two categories based on the transparency of the index to the user.

    Class 1 Search Tools

    • Class 1 search tools, such as AltaVista and Google, completely hide the organization and content of the index from the user.
    • These general-purpose search tools focus on providing relevant search results without revealing details about the index's structure.

    Class 2 Search Tools

    • Class 2 search tools, like Yahoo!, feature subject directories that organize the web content hierarchically.
    • Users can browse and search within these subject directories, which provide a visible and structured representation of the web.

    Search Services

    • Search services offer an abstraction layer over multiple search tools and databases, simplifying web search for users.
    • They broadcast user queries to various search engines and information sources simultaneously.
    • Search services merge the results obtained from these sources, eliminate duplicates, and present the consolidated results as an HTML page with clickable URLs.
